The corpus is built from `tools/memory-efficacy-bench/corpus/findings.json` joined to `tools/memory-efficacy-bench/corpus/gold-labels.json` by `findingId` (skipping 11 unlabeled findings) and serialized as one-record prompt/completion JSONL examples for two framings: a 4-way classification prompt with explicit labels and a binary memory-related prompt with `memory_related` true for all non-`not_memory_related` classes. The split is deterministic by `hash(findingId) % 10`; buckets `0-7` go to `train`, `8` goes to `valid`, and `9` goes to `test`, so synthetic rows no longer cluster after sorting. After the 2026-05-16 synthetic-positive merge (45 minority rows added), counts are 4-way `train: missed_registration=13, recall_miss=18, memory_ignored=15, not_memory_related=371`, `valid: missed_registration=2, recall_miss=2, memory_ignored=2, not_memory_related=37`, `test: missed_registration=2, recall_miss=3, memory_ignored=2, not_memory_related=48`, and binary `train: memory_related=46, not_memory_related=371`, `valid: memory_related=6, not_memory_related=37`, `test: memory_related=7, not_memory_related=48`. The chat corpora mirror the same split totals: 417 train rows, 43 valid rows, and 55 test rows for both 4way-chat and binary-chat. Each row in `gold-labels.json` carries a `source` field (`canonical` for the 470 codex-acp-spark labels; `synthetic` for the 45 codex-acp-deep-synth ones) — use this to filter or weight by provenance.
