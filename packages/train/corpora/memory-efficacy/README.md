The corpus is built from `tools/memory-efficacy-bench/corpus/findings.json` joined to `tools/memory-efficacy-bench/corpus/gold-labels.json` by `findingId` (skipping 11 unlabeled findings) and serialized as one-record prompt/completion JSONL examples for two framings: a 4-way classification prompt with explicit labels and a binary memory-related prompt with `memory_related` true for all non-`not_memory_related` classes. The split is deterministic within each source class by sorting rows by `findingId` and assigning indices `% 10` to `train` (0–7), `valid` (8), and `test` (9); this gives 4-way counts `train: missed_registration=2, recall_miss=8, memory_ignored=4, not_memory_related=366`, `valid: missed_registration=0, recall_miss=0, memory_ignored=0, not_memory_related=45`, `test: missed_registration=0, recall_miss=0, memory_ignored=0, not_memory_related=45`, and binary counts `train: memory_related=12, not_memory_related=366`, `valid: memory_related=1, not_memory_related=45`, `test: memory_related=1, not_memory_related=45`.
