The corpus is built from `tools/memory-efficacy-bench/corpus/findings.json` joined to `tools/memory-efficacy-bench/corpus/gold-labels.json` by `findingId` (skipping 11 unlabeled findings) and serialized as one-record prompt/completion JSONL examples for two framings: a 4-way classification prompt with explicit labels and a binary memory-related prompt with `memory_related` true for all non-`not_memory_related` classes. The split is deterministic per class by sorting rows within each class by `findingId` and assigning per-class indices `% 10` to `train` (0–7), `valid` (8), and `test` (9); this stratifies minority classes across hold-out splits. After the 2026-05-16 synthetic-positive merge (45 minority rows added), counts are 4-way `train: missed_registration=15, recall_miss=19, memory_ignored=16, not_memory_related=366`, `valid: missed_registration=1, recall_miss=2, memory_ignored=2, not_memory_related=45`, `test: missed_registration=1, recall_miss=2, memory_ignored=1, not_memory_related=45`, and binary `train: memory_related=50, not_memory_related=366`, `valid: memory_related=5, not_memory_related=45`, `test: memory_related=4, not_memory_related=45`. Each row in `gold-labels.json` carries a `source` field (`canonical` for the 470 codex-acp-spark labels; `synthetic` for the 45 codex-acp-deep-synth ones) — use this to filter or weight by provenance.
