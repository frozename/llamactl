## Verdict — the K-track LoRA exploration is complete

This is the third independent dataset (K.1 easy gold, K.4 zero-shot adversarial, K.5 adversarial-trained held-out) where the LoRA adapter at `rank=16, num_layers=16, iters=300` is indistinguishable from the base on Qwen3-4B-Instruct-2507. The pattern is consistent:

1. **K.1** (24 easy rows): 100% base, 100% adapter — base saturates with `--jinja`.
2. **K.4** (25 adversarial, base+K.1 adapter): 24% strict / 84-88% name-first, both base and adapter identical to within 1 row.
3. **K.5** (8 held-out adversarial, base+K.5 adapter): 37.5% strict / 50% prefix / 62.5% name-first, **adapter byte-identical to base**.

The adversarial-review devil's advocate persona called this out: repeated no-lift outcomes signal the methodology, not the data, is the issue. At this LoRA configuration the adapter cannot learn anything the chat-template + base model doesn't already encode.

Concrete things to try if returning to the track:
- **Larger rank** (32, 64, 128) — the per-layer parameter budget is the most-suspected ceiling.
- **More layers** (NUM_LAYERS=32 covers ~all attention layers in Qwen3-4B).
- **Higher iters** (1000-3000) — 300 iters on 38 rows is barely 8 epochs.
- **Different base** — Qwen3-8B-Instruct (the production model) instead of 4B. The relevant production failure modes are at 8B, not 4B.
- **Different training framing** — multi-turn rollouts with tool results, instead of single-turn message lists.

But none of these are "obvious wins." The honest interpretation of K.1-K.5 is: **for tool-call grammar tasks where `llama-server --jinja` is available, a small-rank LoRA on the base does not move the needle.** The chat template already encodes the structural information the LoRA would otherwise learn.

## What did move the needle (this session, non-K)

The scorer change (4a6f3ae) is the actual valuable artifact from the K-track:
- strict 24% → prefix 36% revealed that **multi-tool sequential emission** is functionally correct.
- name-first 84-88% revealed that **tool selection is nearly perfect**; the failure surface is argument shapes and ambiguous-intent boundaries.

These are useful signals for production tool-call ergonomics regardless of whether LoRA ever lifts the strict metric.
