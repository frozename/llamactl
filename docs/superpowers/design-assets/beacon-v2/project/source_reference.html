Beacon

A unified design system extracted from two operator tools — llamactl on the web, novaflow on the desktop — refined into one token library, four themes, and a single vocabulary for every surface.

Overview

Two tools. One system.

llamactl needs density — tight tables, terminal affordances, long-running telemetry. novaflow needs breathing room — warm palettes, editorial type, canvas-first work. The system below treats the theme as a variable, not an identity.

v2.0 · frozen Inter · JetBrains Mono CSS custom properties Drop-in stylesheet MIT · Internal

Principles

01

Token in, token out

Every color, spacing step, and radius resolves to a CSS custom property. Swap the theme, swap the feel — nothing downstream re-renders.

02

Semantics before paint

Components never name a color. They reach for surface-2, border-subtle, text-secondary. Status hues stay fixed across themes.

03

Density with breath

Operator surfaces run tight — 11–13px type, 4–8px gaps. Docs and marketing breathe — 19px body, 56–120px section rhythm.

04

Motion as seasoning

Reveal, never perform. Pulses on live data, glows around focus, slow fades on theme switch. Every transition under 400ms.

Getting started

Drop tokens-v2.css into any project. Set data-theme on the root element to switch families. Components in primitives.tsx and patterns.tsx are framework-agnostic — adapt them to your stack.

The activity rail on the left opens the token inspector — live values for every CSS variable in the active theme.

Provenance — where it came from

Every decision traces to one of two repositories — or to the bridge invented during the merge. This is the ledger.

github.com/frozename/llamactl

llamactl — operator baseline

Dense web dashboard for LLM infrastructure. Tight surfaces, long telemetry, disciplined terminal aesthetic.

  • Indigo brand hue→ Sirius theme
  • Dark surface stack (0–4)→ surface tokens
  • Inter + JetBrains Mono→ ui + data type
  • Status-dot convention→ 04.03
  • Stat-card grid→ 05.01
  • Command palette · ⌘K→ 05.03
github.com/frozename/novaflow

novaflow — creative surface

Desktop workspace for flow-based composition. Warm palettes, editorial breathing room, canvas-first interaction.

  • Amber brand hue→ Ember theme
  • Warm neutrals→ Ember surfaces
  • Editorial rhythm→ type scale
  • Generous spacing→ doc tempo
  • Soft focus glows→ button / input
  • Canvas atmosphere→ color composition
new · unified layer

Bridge — the unified layer

What neither repo had, but both needed. Invented during the merge.

  • Theme-as-variable→ 4 families, 1 DNA
  • Clinical light theme→ print & exports
  • Scrubs teal theme→ ops focus
  • Status palette pinned→ cross-theme invariant
  • Motion grammar→ reveal, glow, pulse
